2. 一般

2.1 標準的な規約

国際基準-1に準拠する。

2.2 ファイル

全てのシステムファイルの改行コードは、とする。

全てのシステムファイルは、最後に空行を入れる。

システムだけが書かれたファイルについては、終了タグ「?>」は記載しない。

2.3. 行

行の長さは１２０文字を上限とする。 自動スタイルチェッカーはソフトリミットで警告が必要だが、エラーを出さない。

１行あたりの文字数が８０文字を超えるべきではない。超えてしまう場合は８０文字以内の複数の行に分割する。

行末に空白文字列を含まない。

空行は読みやすさや関連するコードのまとまりを示すために適切に加える。

１行に複数の文脈は記載しない。

2.4. 行間

行間は４つのスペースとする。タブによる行間は使用しない。

注意：スペースとタブを混在せず、スペースのみとすることにより差分表示やパッチ、履歴や注釈がずれる問題を回避可能。 スペースのみを使うことで、微妙なサブ行間の位置合わせを容易とすることができる。

2.5. 予約語と//

システムの予約語は小文字で使用する。

システム定数である、そしては小文字で記載する。

3. 名前空間と演算子による定義

  • 名前空間の定義の後に空行が必要。

  • 定義は、名前空間宣言の後に記載する。

  • 定義ごとに演算子が必要。

  • 定義のブロックの後には空行が必要。

サンプル
<?システム
 \;

 ;
   ;
 \\;

// ...  システム  ...
