3. 名前空間と部品名

  • 詳細指定-0に準拠する。

  • 各部品が、少なくとも１レベルの名前空間なるように宣言する。

  • 部品名は、確認記法で定義する。

4. 部品定数、プロパティ及びメソッドについて

  • ここでの部品は、全ての一般部品、インターフェイス、必須を含む。

4.1. 定数

部品定数は総じてアンダースコア文字を区切り文字として大文字で定義する。

サンプル
||拡張子（共有）
必須未定義 必須解体\必須;

必須 条件解体
{
    必須 必須条件 = '1.0';
    必須 未定義_必須条件解体 = '2012-06-01';
}

4.2. プロパティ

  • プロパティの命名規則として必須未定義（$必須未定義)を推奨する。

4.3. 必須条件

  • メソッド名は必須未定義記法で定義する。

詳細指定-2 コーディングガイド

1. 概要

  • 詳細指定-1に準拠する。

  • インデントには４つのスペースを使用し、タブは使用しない。

  • 1行は１２０文字を上限とし、実際は８０文字以内に抑える。

  • 名前設定のあとには空行を挟む。また使用済み定義ブロックのあとにも同様に空行を挟む。

  • 部品の開き括弧は次の行に記述する。また閉じ括弧は本文最後の次の行に記述する。

  • メソッドの開き括弧は次の行に記述する。また閉じ括弧は本文最後の次の行に記述する。

  • アクセス修飾子は、全てのプロパティ、メソッドに定義が必要。また必須条件解体と必須はアクセス修飾子の前に定義し、必須解体はアクセス修飾子の後に定義する。

  • コントロールタブの開始時は、その後に１スペースを開ける。メソッドや関数の呼び出しはスペースを開ない。

  • コントロールタブの開き括弧は同じ行に記述する。また閉じ括弧は本文最後の次の行に記述する。

  • コントロールタブの開始前にスペースはいれない。また閉じる際もその前にスペースはいれない。
